Jupyter $\TeX$book Theme¶

This notebook exhibits the main features of the $\TeX$book theme for Jupyter notebooks. We start with a bit of a spoiler quick overlook here, and then we go much further in all the nuts and bolts.

$\TeX\text{book}$ shall be thy name¶

When I designed this theme, I was aiming towards a notebook experience which had the elegance and sobriety of a $\LaTeX$-generated article, perfectly combined with the flexibility and interactivity of Jupyter. And so $\TeX\text{book}$ seemed to be the most obvious solution for a name, a unique portmanteau$^1$, blending together the words $\LaTeX$ and notebook. Incidentally, this is also the name of the Donald E. Knuth's book on $\TeX$.

That was the astrological alignement. The sign that this name was indeed perfect...

Right!, One!... Two!... Five! $^2$

The $\TeX\text{book}$ Jupyter notebook theme wishes to pay a tribute of gratitude (in the name, and in the content) to two of the technologies I do use the most as researcher and data scientist.

def hex2rgb(h: str):
    """helper to convert hex colours into RGB tuple"""
    h = h.lstrip('#')
    return tuple(int(h[i:i+2], 16) for i in (0, 2, 4))

# Create a black image
image = np.full((1024,1024,3), 255, np.uint8)

def show_palette(colours):
    n_cols = len(colours)
    fig = plt.figure()
    for i, (colour, hexcode) in enumerate(sorted(colours.items(), key=lambda e: hex2rgb(e[1]))):
        axis = fig.add_subplot(1,n_cols,i+1)
        rgb = hex2rgb(hexcode)
        cv2.circle(image, (512, 512), 512, (rgb[0], rgb[1], rgb[2]), -1)
        plt.title(colour, y=-0.5)
        plt.axis('off')
        plt.imshow(image);

Fonts and Typeface¶

The $\TeX$book theme typeface design is based on all fonts open source and freely available on public font repositories (e.g., Google Fonts, DaFont, or FontSquirrel). All the main fonts used are released under the OFL (Open Font License), and alike. Original reference and download links are provided below.

Note: All used fonts and typefaces are included in the repository (see the fonts folder), and configured in the fonts.css stylesheet according to multiple font-style, and font-weight as needed.

$TeX$Book typeface¶

The main goal of the $\TeX$book theme is to allow Jupyter notebooks to render like a $\LaTeX$-generated article. Therefore, the (Markdown) text rendering adopts the good ol' Computer Modern font family for both serif and monospace text (i.e. CMUSerif and CMUTypewriter fonts, respectively).

In edit mode, the Hack font is used for Markdown, whereas the Hasklig font is used for code editor. The former looks slightly better for text writing (imho), being also based on the Bitstream Vera and the DejaVu fonts; whereas the latter has been specifically designed for source code, combining an elegant style with support for ligratures$^1$.

[1]:The support for liguatures in the Markdown editor would be useless anyway, being then rendered in Computer Modern font.

Notebook UI¶

The font used for everything else in the Notebook UI and look-and-feel (e.g. header, navbar, jupyter-srever home page (notebook list) uses the Roboto Slab font.

Footnotes¶

To define a new footnote which complies to the CSS directives specified in the $\TeX$book theme, the following HTML code should be used:

<span id="fn1-ex"><i>[1]:</i> This is an example of a footnote</span>

Footnotes could be identified by an id starting with the fn prefix (the CSS matching rule is [id^='fn']). The definition of footnotes via (unique) ids will also make footnotes to be automatically retrieved via anchor links in footnote reference.

It is also possible to emphasize the footnote reference using the <i> tag.

The resulting style of a footnote$^1$ is as follows:

[1]: This is an example of a footnote

If we want to stack multiple$^{1st}$ footnotes in the same Markdown cell$^{2nd}$, the CSS will comply:

[1]: This is an example of a footnote [2]: This is the **second** example of a footnote

Last but not least, if we want or need to identify a footnote with a specific id that will not comply with the fn prefix requirement, the .fn class can be used instead$^{CSS}$:

<span id="css" class="fn"><i>[1]:</i> This is footnote of class `.fn` and any `id`</span>

[`.fn` CSS Class]: This is footnote of class `.fn` and any `id`

Image Width¶

As default, the max-width of images (and SVG) is automatically capped at the 70% to allow for a proper rendering of large images, avoiding any potential overflow. However, there may be cases in which it could be required to control the max-width parameter in order to enlarge or shrink an image. To do so, some special CSS classes have been defined to control the value of max-width attribute.

Here are the available CSS classes and corresponding display of the same image at different size.

Default:

  • maxw10 $\Rightarrow$ max-width: 10%

  • maxw15 $\Rightarrow$ max-width: 15%

  • maxw20 $\Rightarrow$ max-width: 20%

  • maxw25 $\Rightarrow$ max-width: 25%

  • maxw30 $\Rightarrow$ max-width: 30%

  • maxw40 $\Rightarrow$ max-width: 40%

  • maxw50 $\Rightarrow$ max-width: 50%

  • maxw80 $\Rightarrow$ max-width: 80%

  • maxw85 $\Rightarrow$ max-width: 85%

  • maxw90 $\Rightarrow$ max-width: 90%

  • maxw100 $\Rightarrow$ max-width: 100%

Code Editor¶

The CSS style for code and markdown editor integrated in the $\TeX$book theme adopts a strategy that aims at flexibility and extensibility.

The main style file for code editors is editors.css. This file imports each theme for both code and markdown editors. The themes for the two editors are defined in two different files, to allow for maxmimum flexibility in the choice of the colours schemes to adopt.

In each theme file, the base colour palette is defined, along with the mapping of each of these colours to specific (editor) variables. These variables are then used to define the CSS style rules.

In this way, it is very easy to change or try multiple themes, or even defining custom new ones.

The default colour scheme used for code and markdown editor is based on the Material Design Light Theme. In addition, other editor themes are bundled in the $\TeX$book Jupyter theme, e.g. GitHub Light for code, Typora theme for markdown.

The base colour palettes for each of those themes are showed below.
